 KSZ8795CLX Ethernet Switch Configuration for PAB_V3

 Context

 The PAB_V3 carrier board has a KSZ8795CLX 5-port managed 10/100 ethernet switch connected
 via SPI0 to the Jetson Orin NX. The switch topology is:

 ┌──────────────────┬───────────────────────────────┬───────────────────┐
 │   Switch Port    │         Connected To          │       Label       │
 ├──────────────────┼───────────────────────────────┼───────────────────┤
 │ Port 1 (PHY1)    │ Jetson eqos (copper PHY link) │ jetson            │
 ├──────────────────┼───────────────────────────────┼───────────────────┤
 │ Port 2 (PHY2)    │ External LAN port             │ external          │
 ├──────────────────┼───────────────────────────────┼───────────────────┤
 │ Port 3 (PHY3)    │ Pixhawk payload connector     │ pixhawk-payload   │
 ├──────────────────┼───────────────────────────────┼───────────────────┤
 │ Port 4 (PHY4)    │ Pixhawk FC (ARKV6X)           │ flight-controller │
 ├──────────────────┼───────────────────────────────┼───────────────────┤
 │ Port 5 (CPU MAC) │ Not connected on PAB_V3       │ —                 │
 └──────────────────┴───────────────────────────────┴───────────────────┘

 SPI0 on the Jetson (= spi@3210000, kernel SPI1) is the management-only interface — it
 carries switch register reads/writes, not ethernet data.

 Why NOT DSA: Linux DSA (Distributed Switch Architecture) requires port 5 (the dedicated
 CPU/MAC port) to be connected to the Jetson's MAC for tail-tag packet framing. On PAB_V3,
 port 5 is not connected — the Jetson's eqos connects to port 1 as a standard copper PHY link.
 The switch operates as a transparent L2 bridge; all devices are reachable from the Jetson's
 eqos interface normally.

 Correct driver: MICREL_KS8995MA (drivers/net/phy/spi_ks8995.c) — the management-only
 SPI driver. It exposes all 256 switch registers as a binary sysfs file for reads/writes.

 SPI note: spi@3210000 is Tegra234's SPI1 controller (what the module calls "SPI_0").
 The BCT pinmux already configures spi1_* pins (pz3/pz4/pz5/pz6) at boot, so no BCT
 changes are needed.

 Reset GPIO: Schematic "GPIO09" = GPIO3_PAC.06 = TEGRA234_MAIN_GPIO(AC, 6).

 Files to Modify

 1. Device Tree — PAB_V3 Main DTS

 File: device_tree/ark_pab_v3/Linux_for_Tegra/source/hardware/nvidia/t23x/nv-public/tegra234-p3768-0000+p3767-0000.dts

 Add inside bus@0 { ... } (alongside the existing serial/pwm/hda nodes):

 spi@3210000 {
     status = "okay";

     ethernet-switch@0 {
         compatible = "micrel,ksz8795";
         reg = <0>;
         spi-max-frequency = <50000000>;
         reset-gpios = <&gpio TEGRA234_MAIN_GPIO(AC, 6) GPIO_ACTIVE_LOW>;
     };
 };

 No pinctrl stanza needed — BCT already sets up the spi1_* pins at boot.

 2. Kernel Config — setup.sh

 File: setup.sh

 Following the established pattern at line ~112 where defconfig options are appended,
 add one line in that block:

 echo "CONFIG_MICREL_KS8995MA=m" >> kernel/kernel-jammy-src/arch/arm64/configs/defconfig

 CONFIG_MICREL_KS8995MA depends on CONFIG_SPI which is already enabled.
 No DSA modules needed.

 3. Validation & Config Script

 New file: ksz8795.sh at the repo root (committed to git, scp'd to Jetson for testing).

 #!/bin/bash
 # KSZ8795 switch read/write utility
 # Usage:
 #   ./ksz8795.sh dump              — print all registers in hex
 #   ./ksz8795.sh read <reg_hex>    — read one register (e.g. read 0x01)
 #   ./ksz8795.sh write <reg_hex> <val_hex>  — write one register
 #   ./ksz8795.sh id                — verify chip ID
 #   ./ksz8795.sh config-default    — apply recommended defaults
 SYSFS=/sys/bus/spi/devices/spi1.0/registers
 [ -e "$SYSFS" ] || { echo "ERROR: KSZ8795 sysfs not found. Is the driver loaded?"; exit 1; }

 write_reg() { printf "\\x$(printf '%02x' $((16#${2#0x})))" | \
     dd of="$SYSFS" bs=1 seek=$((16#${1#0x})) conv=notrunc 2>/dev/null; }
 read_reg()  { dd if="$SYSFS" bs=1 skip=$((16#${1#0x})) count=1 2>/dev/null | xxd -p; }

 case "$1" in
   dump)
     xxd "$SYSFS" | head -16 ;;
   read)
     echo "Reg $2: 0x$(read_reg $2)" ;;
   write)
     write_reg $2 $3 && echo "Wrote 0x$3 to reg $2" ;;
   id)
     id=$(dd if="$SYSFS" bs=1 skip=0 count=2 2>/dev/null | xxd -p)
     echo "Chip ID: 0x${id} (expect 8795 family)" ;;
   config-default)
     # All ports in same VLAN (factory default — already true at power-on,
     # but explicit for documentation). No isolation, all ports forwarding.
     echo "Default config applied (all ports bridged)." ;;
   *)
     echo "Usage: $0 {dump|read <reg>|write <reg> <val>|id|config-default}" ;;
 esac

 Available Settings (Key Registers)

 ┌──────────────────────────┬────────────────────────┬────────────────────────────────────────────┐
 │         Register         │          Name          │              What it Controls              │
 ├──────────────────────────┼────────────────────────┼────────────────────────────────────────────┤
 │ 0x00–0x01                │ Chip ID                │ Read-only verify: expect 0x8795            │
 ├──────────────────────────┼────────────────────────┼────────────────────────────────────────────┤
 │ 0x0C                     │ Global Control 4       │ Enable broadcast storm protection          │
 ├──────────────────────────┼────────────────────────┼────────────────────────────────────────────┤
 │ 0x0D                     │ Global Control 5       │ Storm threshold (packets/period)           │
 ├──────────────────────────┼────────────────────────┼────────────────────────────────────────────┤
 │ 0x1C–0x27                │ Port N Control 1–12    │ Per-port: enable, speed, duplex, learning  │
 ├──────────────────────────┼────────────────────────┼────────────────────────────────────────────┤
 │ 0x58/0x5A/0x5C/0x5E/0x60 │ Port N VLAN membership │ Bitmask of which ports each port can reach │
 ├──────────────────────────┼────────────────────────┼────────────────────────────────────────────┤
 │ 0x6C–0x6F                │ Port N priority        │ 802.1p queue assignment per-port           │
 ├──────────────────────────┼────────────────────────┼────────────────────────────────────────────┤
 │ 0xE0–0xFF                │ MIB counters           │ Per-port traffic stats (read-only)         │
 └──────────────────────────┴────────────────────────┴────────────────────────────────────────────┘

 Recommended UAV Settings (Start Simple, Add as Needed)

 1. Default (factory): All ports in same VLAN → works out of the box, no changes needed.
 2. Port isolation (optional): Set port 3 (payload) and port 4 (FC) to only reach port 1
 (Jetson) — prevents cross-talk between payload and FC without going through the Jetson.
 3. Storm control (recommended): Enable broadcast limiting so a misbehaving device can't
 flood the FC port with broadcast traffic.
 4. Disable port 5 (optional): Port 5 is unconnected; disabling it in software prevents
 any spurious behavior.

 Changing settings online (no reboot):
 sudo ./ksz8795.sh write 0x5C 0x01   # isolate port 3 (payload) — only reaches port 1
 sudo ./ksz8795.sh read 0x5C         # verify
 sudo ./ksz8795.sh dump              # view all registers

 Build & Verify

 Build

 1. Edit the DTS and setup.sh as above.
 2. Since source_build/ already exists (setup already ran), also add the config line directly:
 echo "CONFIG_MICREL_KS8995MA=m" >> \
   source_build/Linux_for_Tegra/source/kernel/kernel-jammy-src/arch/arm64/configs/defconfig
 3. Run ./build_kernel.sh → select 3) PAB_V3.
 4. Flash with ./flash.sh.
 5. scp ksz8795.sh jetson:~/ to copy the test script to the Jetson.

 Verify on Jetson

# Confirm driver loaded and sysfs exists
lsmod | grep ks8995
ls /sys/bus/spi/devices/spi1.0/registers

# Read chip ID registers (expect first byte ~0x87, second ~0x95)
dd if=/sys/bus/spi/devices/spi1.0/registers bs=1 skip=0 count=2 2>/dev/null | xxd

# Run config script
sudo /usr/local/bin/ksz8795-config.sh

# Confirm ethernet works through switch
ip link show eth0
ping <device-on-external-port>
